## Project Description

This repository contains a vulnerability assessment performed in a personal lab environment.

The purpose of this project is to demonstrate practical skills in vulnerability analysis, web application security testing, and technical reporting. The assessment was conducted following a structured approach similar to what is used in real-world cybersecurity engagements.

---

## Scope

- Target: Vulnerable web application deployed in a local lab
- Environment: Virtual machines (Kali Linux + test web server)
- Testing approach: Black Box
- Methodology reference: OWASP Testing Guide

---

## Tools Used

- Nmap (port scanning and service enumeration)
- Nikto (web server vulnerability scanning)
- Burp Suite (manual web testing)
- Hydra (authentication testing)
- Manual testing techniques

---

## Activities Performed

- Network reconnaissance
- Service enumeration
- Web application testing
- Identification of security vulnerabilities
- Risk evaluation
- Documentation of findings
- Remediation recommendations

---

## Vulnerabilities Identified

During the assessment, several vulnerabilities were identified, including:

- SQL Injection
- Cross-Site Scripting (XSS)
- Weak authentication mechanisms
- Exposed services and open ports

Each vulnerability is documented in the full report with:
- Technical description
- Impact analysis
- Proof of Concept
- Risk level
- Recommended mitigation

---

## Report

The full vulnerability assessment report is included in this repository.

The report is structured to reflect a professional cybersecurity assessment, including technical details and remediation guidance.

## Disclaimer

This assessment was conducted in a controlled laboratory environment for educational and portfolio purposes only.  
No real organizations or production systems were involved.
